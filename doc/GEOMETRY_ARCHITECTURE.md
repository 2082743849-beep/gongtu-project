# 空间几何实验室架构决策

状态：已采纳
版本：1.0
日期：2026-06-29

## 1. 目标

在公途现有 Web、FastAPI、SQLite 和 Electron 架构中增加空间几何实验室，支持：

1. 参数化创建基础立体与积木组合体。
2. 旋转、缩放、正投影视图和顶点标记。
3. 使用可移动、可倾斜的无限平面切割模型。
4. 精确计算并高亮截面，而不只是做视觉遮挡。
5. 将题目图片和文字转换为可校验、可人工修正的模型草稿。
6. 让网页端和 Electron 端复用同一页面与后端服务。

本模块不是通用 CAD，也不承诺从任意单张图片无歧义恢复真实三维物体。首要目标是服务考公空间几何和工图推理题。

## 2. 核心架构决策

### 2.1 确定性几何与 AI 分离

系统分成两条职责链：

- **AI 建模链**：图片和文字 → Geometry JSON 草稿 → 不确定性标记。
- **几何求解链**：经过确认的 Geometry JSON → Three.js 模型 → 截面、投影和答案。

AI 不直接生成可执行 JavaScript，不直接输出最终数学答案，也不绕过 Schema 和几何校验。题目答案必须由确定性几何算法计算或由人工标准答案验证。

### 2.2 参数模型优先于自由网格

第一阶段只接受可解释的参数模型：

- 长方体、正方体
- 三棱柱、三棱锥
- 圆柱、圆锥、球
- 单位立方体坐标阵列
- 上述模型的平移、旋转、缩放与受控布尔组合

自由网格导入属于后续能力，不进入首版验收范围。参数模型更容易校验、修改、测试和复现题目答案。

### 2.3 Geometry JSON 是唯一跨层协议

浏览器、后端、数据库和 AI 只通过版本化 Geometry JSON 交换模型。概念结构如下：

```json
{
  "schemaVersion": "1.0",
  "units": "relative",
  "objects": [
    {
      "id": "cube-1",
      "type": "box",
      "dimensions": {"width": 1, "height": 1, "depth": 1},
      "transform": {
        "position": [0, 0, 0],
        "rotation": [0, 0, 0],
        "scale": [1, 1, 1]
      },
      "appearance": {"color": "#f5f1e8", "opacity": 1}
    }
  ],
  "labels": [],
  "cutPlane": null,
  "uncertainties": []
}
```

正式 Schema 将在 QDB-001 单独定义。本示例只确定边界，不提前锁死字段细节。

## 3. 模块边界

| 模块 | 负责 | 不负责 |
|---|---|---|
| 实验室页面 | 布局、参数控件、状态反馈、二维截面面板 | 几何数学和 AI 请求细节 |
| 场景控制器 | 相机、灯光、轨道控制、渲染循环、资源释放 | 题库持久化 |
| 模型工厂 | Geometry JSON 到 Three.js 对象 | 猜测图片中的隐藏结构 |
| 切面引擎 | 平面表示、求交、排序、截面轮廓与面积 | AI 识图 |
| 投影引擎 | 正交相机视图和轮廓展示 | 透视图片反推 |
| Schema 校验器 | 类型、范围、引用和版本校验 | 判断题目答案 |
| 题库 API | 题目、模型、答案、解析和验证状态 | 浏览器渲染 |
| AI 适配器 | 多模态请求、结构化输出和错误归一化 | 执行 AI 输出代码 |
| 建模任务服务 | 状态机和 WebSocket 进度 | 长期保存密钥 |
| 管理确认页面 | 预览、修改、不确定项确认和入库 | 自动批准模型 |

模块之间禁止共享可变全局状态。Three.js 对象不得直接写入数据库；数据库只保存协议数据和元数据。

## 4. 数据流

### 4.1 手工参数建模

```text
用户选择基础模型
  → 页面生成参数
  → Schema 校验
  → 模型工厂
  → Three.js 场景
  → 切面引擎
  → 3D 高亮 + 2D 截面 + 数值信息
```

### 4.2 正式题库加载

```text
题目 API
  → 题目文字、原图、Geometry JSON、标准答案
  → Schema 校验
  → 模型工厂
  → 学生交互
  → 作答 API
  → 结果与解析
```

### 4.3 AI 辅助建模

```text
管理员上传图片并输入文字
  → 文件安全校验
  → 创建建模任务
  → WebSocket 推送进度
  → 多模态模型生成 Geometry JSON 草稿
  → Schema 校验
  → 几何语义校验
  → 预览并显示不确定项
  → 管理员确认或修正
  → 正式入库
```

任何校验失败都会停止入库，并返回可理解的错误，不得静默修补成另一个模型。

## 5. 切割与截面定义

“刀”在数学上表示无限平面：

```text
normal · point + constant = 0
```

交互层可以通过三种方式定义它：

1. 法向量加平面位置。
2. X、Y、Z 倾角加距离。
3. 三个不共线点。

显示层可以画有限半透明矩形帮助观察，但计算必须使用无限平面。多面体截面通过“边与平面求交 → 去重 → 投影到平面局部坐标 → 环形排序”得到。曲面体必须使用对应解析方案或受控网格误差方案，并明确精度。

视觉裁剪与数学截面是两个功能：视觉裁剪负责看见内部，数学截面负责给出真实轮廓、顶点、边数和面积，两者不可混为一谈。

## 6. 数据与接口原则

- 所有模型带 `schemaVersion`，读取旧版本时执行显式迁移。
- 正式题目保存原图引用、题干、模型、标准答案、解析、验证状态和更新时间。
- AI 草稿与正式模型分开保存，未确认草稿不能出现在学生题库。
- API 使用 Pydantic 校验；浏览器仍需做即时输入校验。
- WebSocket 只传任务状态和小体积结构化消息，大图片通过普通上传接口传输。
- AI 服务密钥只从 `.env` 读取，响应和日志不得包含密钥。

## 7. 失败与降级策略

| 失败场景 | 系统行为 |
|---|---|
| WebGL 不可用 | 显示明确说明，不进入空白页面 |
| 模型参数非法 | 阻止渲染并定位具体字段 |
| 平面与模型不相交 | 显示“暂无截面”，不作为错误 |
| 平面经过顶点或与面重合 | 使用容差规则并提示退化情况 |
| AI 服务不可用 | 保留上传内容，允许稍后重试或手工建模 |
| AI 无法确定遮挡结构 | 写入 `uncertainties`，要求人工选择 |
| WebSocket 断开 | 使用任务状态接口恢复，不重复创建任务 |
| 旧题模型版本 | 迁移成功后加载；失败则停止并提示管理员 |

## 8. 主要风险与控制

| 风险 | 影响 | 控制措施 |
|---|---|---|
| AI 从单视图猜错隐藏结构 | 错题、错误答案 | 不确定性标记、人工确认、AI 不参与求解 |
| 浮点误差导致重复点或开口截面 | 截面闪烁、面积错误 | 统一 epsilon、退化样例测试、局部二维排序 |
| CSG 对复杂模型性能不足 | 页面卡顿 | 参数模型优先、复杂度限制、后台预计算 |
| CDN 或网络依赖不可用 | 桌面端离线失败 | 固定版本并打包本地依赖 |
| 原生单页继续膨胀 | 难维护 | 空间几何脚本和样式独立目录、模块化加载 |
| 上传文件滥用 | 安全与存储风险 | 类型、大小、路径、配额和清理策略 |
| 新功能破坏旧模块 | 回归 | 独立路由、冒烟测试、PR 与 CI 门禁 |

## 9. 阶段验收标准

### M0：工程治理

- 规则只有一个明确优先级。
- 任务、状态、提交和推送可追踪。
- 架构决策、依赖和 CI 基线确定。

### M1：基础实验室

- 七类基础模型可通过参数创建。
- 可旋转、缩放、复位并显示坐标辅助。
- 非法参数有明确错误，WebGL 失败可降级提示。

### M2：切面内核

- 无限平面可移动、倾斜并可由三点确定。
- 典型多面体和旋转体能显示真实截面。
- 3D 高亮、2D 轮廓和数值信息一致。
- 退化与浮点边界测试通过。

### M3：组合与视图

- 积木坐标阵列能够还原固定样例。
- 六个正交方向可观察或生成轮廓。
- 组合体切面固定样例与人工答案一致。

### M4：题库

- 模型协议版本化且前后端都能校验。
- 草稿、正式题和答案状态可区分。
- 管理员可参数化编辑、预览并录入样例题。

### M5：AI 辅助建模

- 图片和文字能生成结构化草稿与不确定项。
- 错误或恶意 AI 输出无法绕过校验。
- WebSocket 中断后能恢复任务状态。
- 未经人工确认的模型不能进入正式题库。

### M6：发布

- Web 与 Electron 冒烟测试通过。
- CI 全绿，复杂样例性能达到发布清单要求。
- README、部署说明和管理员操作说明齐全。

## 10. 非目标

首版明确不做：

- 从任意照片自动生成高精度写实网格。
- 替代 Blender、AutoCAD 或完整三维编辑器。
- 无人工确认地批量发布 AI 生成题目。
- 物理破碎、材料仿真或制造级尺寸公差。
- 在手机端提供与桌面端相同复杂度的建模操作。

这些边界保证第一版先把考题所需的“看得见、切得准、算得对”完成。
